# Narration Guide

Reference for writing effective explainer video narration scripts.

## Length

- **Target:** 150-200 words for 60-90 seconds
- **Speaking rate:** ~2.3 words/second (professional narration)
- **Shorter is better** — every word competes with the visual demo

## Structure

### 1. Hook (5-10 seconds, ~15-25 words)
State what the tool is and what it does in one clear sentence.

> "This is the IIEF Digital Questionnaire — a browser-based tool that turns patient responses into HL7 files ready for import into Genie."

### 2. Workflow Context (3-5 seconds, ~10-15 words)
How it fits into existing processes. Reassure the viewer nothing changes.

> "It integrates directly into your existing clinical workflow — no changes to how your practice operates."

### 3. Transition (2-3 seconds)
Bridge to the demo.

> "Let's see it in action."

### 4. Demo Walkthrough (40-50 seconds, ~90-120 words)
Narrate what the viewer sees on screen. Keep sentences short. Match timing to the actual demo actions.

Tips:
- Describe actions as they happen: "The patient enters their details"
- Call out notable features: "The postcode automatically fills in the state"
- Name specific UI elements viewers can see
- Group related actions: don't narrate every single click

### 5. Value Propositions (10-15 seconds, ~25-35 words)
Key benefits. Use concrete claims.

> "Everything runs locally in the browser. No patient data leaves the device. No accounts, no subscriptions."

### 6. Call to Action (3-5 seconds)
End with confidence.

> "It's ready to use today."

## Writing Style

- **Short, clear sentences.** Average 10-15 words per sentence.
- **Active voice.** "The system calculates scores" not "Scores are calculated by the system"
- **Present tense.** "The patient enters" not "The patient will enter"
- **No jargon in narration** unless the audience expects it (clinical terms for medical, technical terms for developers)
- **No filler phrases.** Cut "basically", "essentially", "as you can see"
- **Australian English** if targeting AU market (colour, organisation, etc.)

## Timing Alignment

The narration must be timed to match the demo recording + slide overlays:

```
0-10s   : Intro (narrate hook + context while intro/title slides show)
10-22s  : Demographics entry (narrate form filling)
22-26s  : Stat slide (narrate feature highlight)
26-50s  : Main interaction (narrate the core flow)
50-56s  : Section slide (narrate capability)
56-65s  : Results/output (narrate what viewer sees)
65-75s  : Value props + outro (narrate benefits while outro slides show)
```

## Example (177 words, ~77 seconds)

```
This is the IIEF Digital Questionnaire — a browser-based tool that turns
patient responses into HL7 files ready for import into Genie.

It integrates directly into your existing clinical workflow — no changes
to how your practice operates.

Let's see it in action.

The patient enters their details — name, date of birth, Medicare number,
and address. The postcode automatically fills in the state.

Next, the fifteen IIEF questions. Each one is colour-coded by clinical
domain — erectile function, orgasmic function, sexual desire, intercourse
satisfaction, and overall satisfaction. The patient simply selects their
answer for each question.

Once all questions are answered, we click Submit and Calculate Scores.

The results screen shows each domain score alongside clinical
interpretation — from normal function through to severe dysfunction.
Scores are compared against published control means, so you can see at a
glance where the patient sits.

From here, one click downloads a standards-compliant HL7 file with the
PDF report embedded. This imports directly into Genie — no manual data
entry, no transcription errors.

Everything runs locally in the browser. No patient data leaves the
device. No accounts, no subscriptions. It's ready to use today.
```
